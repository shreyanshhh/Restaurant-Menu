Preemptive warning to prevent food allergy

User briefing

  • When ordering food online, people sometime accidentally order food items that they might be allergic to.
  • When ordering through our app, if you add items that you are allergic to, then we will give you a warning to reduce order mistakes.
  • When user registers, we will get their allergen information.
  • We provide clean UI for finding restaurant and adding food item to cart.
  • If user adds food item that they are allergic to, then they are presented with warning. They can also disregard the warning for cases when they are ordering for their friends.
  • After placing order, user can also track order status in the application

Scenario Tasks

  • Profile creation - User creates profile
  • Restaurant search - User finds a restaurant
  • Add items to cart - User adds item to cart
  • Order monitoring - User orders food and monitors its status

Observations

  • The users were a little confused with the different profile creation steps and needed briefing on how consumer and restaurant owner profiles are different.
  • Some users pointed out that restaurant owners might find it difficult to go through exhaustive lists of ingredients. Sorting them into categories might seem helpful.
  • Some users were concerned about the privacy issues and giving down terms and conditions that are GDPR complaint can make them believe that the app is GDPR complaint.

Prototype iteration

  • The main difference was in the navigation of different pages across the app, back buttons were added in the final design to help user easily navigate to previous visited page.
  • Within the Restaurant menu different categories tab has been added above the search bar to increase the flexibility.
  • The warning sign has been updated to be more user friendly
  • The order page now displays the order quantity and price of each ordered item.
