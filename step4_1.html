Trial sketches - Restaurant search

Good features

  1. The user can scroll up and down thanks to the scroll view layout feature embedded inside the application.
  2. The search view button is right above the restaurant scroll view list. User have more than one way to search.
  3. The photo of the dish is obvious to see

Bad features

  1. Can navigate one menu at a time and all may not be available
  2. The restaurant list build from time to time. Constant updates.
  3. A typical user interaction may feel Awkward at first.

Trial sketches - Add items to cart

Good features

  1. Navigation bar at the bottom to go to different screeens
  2. Search bar to search within the menu
  3. Shows warning on adding allergen food item
  4. Next button on top right to place order

Bad features

  1. No button to go back to home page
  2. No indication of how many items are added in the menu
  3. Buttons on warning modal might be misleading to user.

Trial sketches - Order monitoring

Good features

  1. The navigation bar gives user flexibility to move accross home, order, cart and account settings.
  2. The order page gives information regarding whether the deliver partner is assigned or not.
  3. Order list gives accurate information of the items ordered.
  4. Once the delivery partner is assigned his/her name along with contact information and location is shared with the user.

Bad features

  1. The design lacks navigation buttons like back if the user wants to go back to a previous page.
  2. The design lacks displaying of quantity and price of each order item.
